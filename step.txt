PHASE 1: PROJECT SETUP (STARTING THE PROJECT)

Step 1: Create Virtual Environment

* We create a virtual environment to keep our project separate from other projects.
* This avoids package conflicts.

Command:
python -m venv venv

* Then activate the environment.

---

Step 2: Create template.py

* This file helps us automatically create folders and files.
* Instead of manually creating everything, we run:

python template.py

* It builds the basic project structure.

---

Step 3: Create Constants File
Location: src/cnnClassifier/constants/**init**.py

* This file stores important file paths like:

  * config.yaml (project settings)
  * params.yaml (model parameters)

Why?

* So we don’t hardcode paths again and again.

---

Step 4: Create Logger
Location: src/cnnClassifier/**init**.py

* Logger is used to track what is happening in the project.
* It helps in debugging and monitoring.

Example log:
[INFO] Data ingestion started
[INFO] Data downloaded successfully

---

Step 5: Create main.py

* This is the main file from where the project runs.
* We use it to test logger first.

Run:
python main.py

---

Step 6: Install Requirements

* Add all required libraries in requirements.txt
* Install them using:

pip install -r requirements.txt

---

==================================================

PHASE 2: DATA INGESTION (GETTING THE DATA)

What is Data Ingestion?

* It means downloading data from a source and storing it locally.

---

Step 1: Create Utility File
Location: src/cnnClassifier/utils/common.py

* This file contains helper functions like:

  * create folders
  * read YAML files
  * save/load files

---

Step 2: Update config.yaml
Location: config/config.yaml

* This file stores project settings.

Example:

artifacts_root: artifacts

data_ingestion:
root_dir: artifacts/data_ingestion
source_URL: dataset link
local_data_file: artifacts/data_ingestion/data.zip
unzip_dir: artifacts/data_ingestion

Explanation:

* root_dir → where data will be stored
* source_URL → where data comes from
* local_data_file → where zip file is saved
* unzip_dir → where data is extracted

---

Step 3: Update params.yaml

* This file stores model parameters.

Example:

IMAGE_SIZE: [224, 224, 3]
BATCH_SIZE: 16
EPOCHS: 1
LEARNING_RATE: 0.01

Difference:

* config.yaml → project setup
* params.yaml → model training settings

---

Step 4: Create Research Notebook
Location: research/data_ingestion.ipynb

* We write and test code here before moving to main project.

---

Step 5: Create Entity Class
Location: src/cnnClassifier/entity/config_entity.py

* Entity class stores structured data.

Example:
class DataIngestionConfig:
pass

---

Step 6: Update Configuration Manager
Location: src/cnnClassifier/config/configuration.py

* Add function:
  get_data_ingestion_config()

* This function reads config.yaml and returns values.

---

Step 7: Create Component
Location: src/cnnClassifier/components/data_ingestion.py

* This file contains actual logic:

  * download data
  * unzip data

---

Step 8: Create Pipeline
Location: src/cnnClassifier/pipeline/stage_01_data_ingestion.py

* Pipeline connects all steps together.

---

Step 9: Update main.py

* Add code to run data ingestion stage.

---

Output:

* Data will be downloaded and extracted in artifacts folder.

SIMPLE ARCHITECTURE UNDERSTANDING

params      → model settings (epochs, batch size)
entity      → data structure
